# 要件定義 - GitHub Star History メーカー

## 1. 概要
好きなGitHubリポジトリを複数選択して、スター数を時系列でグラフ化できるブラウザツール。
GitHub Pages で公開可能な HTML+JavaScript+CSS の静的サイトとして構築する。

## 2. 機能要件

### 2.1 リポジトリ指定
- ユーザーがGitHubリポジトリのURL（または `owner/repo` 形式）を複数入力できる。
- 入力されたリポジトリごとにスター履歴を取得する。

### 2.2 期間指定
- デフォルトは「全期間（リポジトリ作成日〜現在）」を表示する。
- オプションで集計開始日と終了日を指定できる。
- リポジトリ作成日前のデータはグラフに表示しない。

### 2.3 グラフ表示・加工
- リポジトリごとに色分けして可視化する。
- グラフの種類を選択できる：
  - 折れ線グラフ（デフォルト、プロットなし）
  - 棒グラフ（積み上げ）
  - 棒グラフ（積み上げなし）
- 表示範囲の調整：
  - 横軸（時間）の最小値・最大値を変更可能。
  - 縦軸（スター数）の最小値・最大値を変更可能。
- 色の変更：
  - リポジトリごとに線の色や棒の色を自由に変更可能。

### 2.4 GitHub Token 設定
- APIレート制限を回避するため、ユーザーが自身の GitHub Personal Access Token (PAT) を設定できる。
- 設定画面を用意し、そこで入力・保存（LocalStorage）を行う。
- Token取得の初心者向けに、詳細なガイド（手順・リンク・権限設定など）を表示する。
- 外部サーバーへの送信は行わず、ブラウザ内でのみ保持することを明記する。

## 3. 非機能要件
- GitHub API v3 (REST API) を使用する。
- グラフライブラリとして Chart.js を使用する。
- GitHub Pages で公開可能な構成にする。
- レスポンシブデザイン（PCメインだがスマホでも閲覧可能）。
