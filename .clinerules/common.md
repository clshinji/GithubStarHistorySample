# プロジェクト全体のルール

## 言語設定

- すべての回答は日本語で行ってください。
- 技術的な用語は、適切な日本語訳を使用し、必要に応じて元の英語を併記してください。
- コード内のコメントやドキュメントも可能な限り日本語で記述してください。
- あなたの一人称は「クラインちゃん」を使用してください。

## 開発方針

- SDD（仕様駆動開発）で実装しましょう。
  - ただし、簡易的な、簡単なタスクであれば、SDD（仕様駆動開発）でなくてもOKです。
- SDDの「要件」、「設計」、「タスクリスト」は `spec` ディレクトリに記録します。
- プロジェクトごとに、サブディレクトリを作成するので `spec/001_xxxx` のようにサブディレクトリを作成して、要件・設計・タスクリストのファイルを記録します。
- 要件として `requirements.md` を、設計として `design.md` を、タスクリストとして `task.md` を作成します。
- 作業が進んだら、こまめに `task.md` に記録してね！

## このプロジェクトでの追加指示

- Google App Script (GAS) のコードを作成します。
- GAS API を使うために **clasp** を使っています。
- コードの修正ができたら `clasp push` コマンドで、オンラインの GAS のコードに反映させます。その後、私（開発者）が GAS での動作を確認します。
- `npm run push:prod` コマンドは本番に反映するためのコマンドなので、私（開発者）の許可を得てから実行してください。
